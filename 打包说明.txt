# 重要：重新打包说明

## 问题原因
你遇到的问题是：摄像头窗口打开了，但控制台显示"未找到摄像头"并卡住。

这是因为：
1. 主进程成功打开摄像头（右侧窗口显示）
2. multiprocessing子进程启动时重新导入主模块
3. 子进程也尝试打开摄像头，但摄像头已被占用
4. 子进程失败导致整个程序卡住

## 已修复
已修改 mario.py，添加子进程检测逻辑：
- 子进程不会执行main()函数
- 只有主进程打开摄像头和运行游戏

## 重新打包步骤

```powershell
# 1. 清理旧文件
cd g:/All_Code/Codebuddy/game/mario-handgesture-game
Remove-Item -Recurse -Force build, dist -ErrorAction SilentlyContinue

# 2. 重新打包
.\一键打包.bat

# 3. 等待10-30分钟

# 4. 测试新的exe
cd dist
.\MarioHandGesture.exe
```

## 预期结果
新的exe应该能够：
1. ✅ 显示摄像头窗口（左侧或右侧）
2. ✅ 显示马里奥游戏全屏窗口
3. ✅ 通过手势控制游戏
4. ✅ 不再卡在摄像头检测
